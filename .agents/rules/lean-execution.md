# Lean Execution

- Do not explain what you're about to do before doing it. Just do it.
- Do not summarize changes after making them, unless explicitly asked, or unless it's the PROJECT_RECAP.md update (see session-recap.md).
- Do not restate the user's request back to them.
- Do not print full file contents when a diff/patch is enough.
- Pick the simplest solution that satisfies the requirement. Do not add abstraction, config options, error handling, or edge-case coverage that wasn't asked for.
- No commented-out code, no placeholder TODOs unless requested.
- If there's one obvious correct approach, take it — don't list alternatives unless asked.
- Ask a clarifying question only if truly blocked; otherwise pick the sensible default, note the assumption in one line, and proceed.
- Final response format: what changed (1-3 lines max) + anything the user needs to do. Nothing else.
